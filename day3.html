Why Semantic HTML?

  • Conveys meaning & structure to browsers and developers
  • Improves Accessibility for assistive technologies
  • Better SEO - search engines understand content hierarchy
  • Easier to maintain and understand code
Slide 2 / 30

Common Semantic Tags in HTML5

  • <header> - Introductory content or site branding
  • <nav> - Navigation links
  • <main> - Main page content
  • <article> - Independent content piece, like blog posts
  • <section> - Thematic grouping of content
  • <aside> - Sidebar or tangential content
  • <footer> - Footer information or links

Slide 4 / 30

Div <div> Overview

  • Generic container with no semantic meaning
  • Used mainly for styling or grouping elements
  • Often overused instead of semantic tags
  • Does not improve accessibility or SEO
Slide 5 / 30

Span <span> Overview

  • Generic inline container
  • Used for styling small parts of text or grouping inline content
  • Does not add any semantic meaning
  • Useful for inline CSS or JS targeting
Slide 6 / 30

Block vs Inline Elements Recap

  • Block elements: Take full width, start on new line (e.g., <div>, <p>, <h1>)
  • Inline elements: Take only required width, flow within text (e.g., <span>, <a>, <img>)
Slide 7 / 30

Why Use Semantic Tags Instead of Divs?

  • Improves document readability
  • Helps Screen Readers understand context
  • Assists search engines for better ranking
  • Helps with easier front-end maintenance
Slide 8 / 30

Accessibility Basics: Importance

  • Ensures web content is usable by people with disabilities
  • Compliance with legal standards (WCAG, ADA)
  • Improves usability for everyone
  • Enables screen readers and assistive technology to communicate content
Slide 9 / 30

Using alt Attribute on Images

  • Describe image content or function
  • Essential for screen readers
  • Empty alt="" for decorative images
  • Example:

Example:

<img src="logo.png" alt="Company logo">
Slide 10 / 30

Using label for Form Inputs

  • Associates text with form controls
  • Improves form accessibility
  • Use <label for="id"> to link label to input
  • Example:

Example:

<label for="email">Email:</label>
<input type="email" id="email" name="email">
Slide 11 / 30

ARIA Attributes Overview

  • ARIA = Accessible Rich Internet Applications
  • Enhance accessibility when native HTML lacks features
  • Common attributes: aria-label, aria-hidden, aria-describedby
  • Use ARIA cautiously, don't replace native HTML semantics
Slide 12 / 30

Example: Using aria-label

<button aria-label="Close menu"><span>X</span></button>

This gives screen readers a descriptive name for the button.

Slide 13 / 30

Example: Hiding Elements From Screen Readers

<div aria-hidden="true">Decorative content</div>

This prevents assistive tech from reading irrelevant content.

Slide 14 / 30

Semantic HTML & Accessibility Best Practices

  • Use semantic HTML first
  • Provide descriptive alt text for images
  • Label all form controls
  • Use ARIA when necessary to fill gaps
  • Ensure keyboard navigability

Activity: Accessibility Checklist

For your semantic page, ensure you:

  • Add descriptive alt text to images
  • Use label tags for form fields
  • Ensure keyboard accessibility
  • Check color contrast

Try testing with a screen reader if possible.

Interview Question 1

Q: What is the difference between <section> and <article> in HTML5?

A: article represents a standalone piece of content suitable for syndication; section groups related content thematically but may not be standalone.

Slide 18 / 30

Interview Question 2

Q: Why should you avoid using <div> tags excessively?

A: Because div tags lack semantic meaning, overusing divs can reduce accessibility, SEO benefits, and make code harder to maintain.

Slide 19 / 30

Interview Question 3

Q: What is the purpose of the alt attribute on images?

A: It provides alternative text for screen readers when images cannot be seen or loaded, improving accessibility.

Slide 20 / 30

Interview Question 4

Q: When should you use ARIA attributes?

A: Use ARIA to fill in gaps where native HTML elements do not provide sufficient semantic meaning or accessibility.

Slide 21 / 30

Interview Question 5

Q: What is the difference between <span> and <div>?

A: span is an inline element usually used for small pieces of text; div is a block-level container for grouping larger sections.
