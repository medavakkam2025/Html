Introduction to HTML5 Advanced Tags

HTML5 introduced new semantic and interactive tags that allow better structuring and dynamic content on web pages.

We will explore these modern tags and how they enhance user experience.

Slide 2/30

The <canvas> Element

The <canvas> element provides a drawable region, used for rendering graphics, animations, and game content via scripting.

It is a bitmap area where you can use JavaScript to draw shapes and images dynamically.

Slide 3/30

Basic Canvas Example

Example code to create a blue rectangle on canvas:

<canvas id="myCanvas" width="200" height="100"></canvas>

<script>
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');
ctx.fillStyle = 'blue';
ctx.fillRect(0, 0, 200, 100);
</script>

Slide 5/30

The <details> and <summary> Elements

The <details> element creates a disclosure widget which users can open and close.

The first child <summary> acts as the visible heading or label for the toggled content.

Slide 6/30

Example of <details> and <summary>

<details>
  <summary>More Info</summary>
  <p>This content is initially hidden but can be shown by clicking the summary.</p>
</details>
Slide 7/30

Responsive Images Importance

Responsive images help deliver the best resolution and size depending on device screen size and resolution, reducing load times and improving UX.

Slide 8/30

The <picture> Element

Allows providing multiple image sources for different device widths or pixel densities with media queries.

<picture>
  <source srcset="small.jpg" media="(max-width: 600px)">
  <source srcset="medium.jpg" media="(max-width: 1200px)">
  <img src="large.jpg" alt="Example image">
</picture>
Slide 9/30

The srcset Attribute

Used in <img> for listing multiple image sizes to allow the browser to pick the best.

<img src="default.jpg" 
     srcset="small.jpg 500w, medium.jpg 1000w, large.jpg 1500w" 
     sizes="(max-width: 600px) 500px, 1000px" alt="Responsive Image">
Slide 10/30

Web Storage Overview

Web Storage allows client-side data to be stored persistently (localStorage) or per session (sessionStorage).

  • localStorage: persists until explicitly deleted
  • sessionStorage: cleared on tab/window close
Slide 11/30

Using localStorage: Save Data

localStorage.setItem('key', 'value');

This saves information under "key".

Slide 12/30

Using localStorage: Retrieve Data

const data = localStorage.getItem('key');
console.log(data); // outputs stored value
Slide 13/30

Using sessionStorage

sessionStorage.setItem('sessionKey', 'sessionValue');

const sessionData = sessionStorage.getItem('sessionKey');
console.log(sessionData);

Slide 15/30

The <template> Element

Defines HTML fragments that are not rendered immediately but can be cloned and inserted dynamically via JavaScript.

<template id="myTemplate">
  <div class="card">Content here</div>
</template>
Slide 16/30

Using <template> with JavaScript

const template = document.getElementById('myTemplate');
const clone = document.importNode(template.content, true);
document.body.appendChild(clone);
Slide 17/30

The <output> Element

Represents the result of a calculation or user action, usable for displaying output dynamically.

Example:

<form oninput="result.value=parseInt(a.value)+parseInt(b.value)">
  <input type="number" id="a" value="0">
  <input type="number" id="b" value="0">
  <output name="result" for="a b">0</output>
</form>
Slide 18/30

Canvas Basic Shapes

  • Rectangle: fillRect(), strokeRect()
  • Circle: arc()
  • Path: beginPath(), lineTo(), stroke()
Slide 19/30

Canvas Drawing Example: Red Circle & Text

const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

ctx.fillStyle = 'red';
ctx.beginPath();
ctx.arc(100, 75, 50, 0, Math.PI * 2);
ctx.fill();

ctx.font = '18px Arial';
ctx.fillText('Red Circle', 60, 150);

Slide 22/30

Demo: Saving User Input with localStorage

Example snippet to save and retrieve a user's name:

const input = document.getElementById('nameInput');
const savedName = localStorage.getItem('name');

if(savedName) {
  input.value = savedName;
}

input.addEventListener('input', () => {
  localStorage.setItem('name', input.value);
});

Accessibility Considerations

  • Use semantic HTML elements
  • Ensure <canvas> content is accessible or provide alternatives
  • Label any inputs clearly
  • Avoid hiding important info with <details> without good reason
