What is HTML?

HTML (HyperText Markup Language) is the standard markup language used to create web pages.

It structures content for the web browsers to display.

Slide 2 of 30

Structure of an HTML Document

Basic components:

  • <!DOCTYPE html> - declares document type
  • <html> - root element
  • <head> - metadata, title, links to CSS/JS
  • <body> - visible content
Slide 3 of 30

Headings in HTML

There are 6 levels of headings: <h1> to <h6>.

<h1> is the highest level heading and <h6> the lowest.

Slide 4 of 30

Paragraphs & Formatting Tags

  • <p> - paragraph
  • <strong> - bold
  • <em> - italic
  • <u> - underline
  • <br> - line break
Slide 5 of 30

Links and Images

  • <a href="url"> - hyperlink
  • <img src="image.jpg" alt="description"> - image

Link example: <a href="https://www.example.com">Example</a>

Image example: <img src="pexels-photo.jpg" alt="Sample Image">

Slide 6 of 30

Lists in HTML

  • <ul> - unordered list
  • <ol> - ordered list
  • <dl> - description list

Example: <ul><li>Item</li></ul>

Slide 7 of 30

Attributes in HTML

Attributes provide additional information about elements.

  • Placed inside the opening tag
  • Format: name="value"
  • Common attributes: id, class, style, href, src
Slide 8 of 30

Comments in HTML

Used to leave notes within the code.

Syntax: <!-- Comment goes here -->

Slide 9 of 30

Block vs Inline Elements

Block elements: Take full width available, start on new lines.

Examples: div, p, ul, h1

Inline elements: Take only required width, flow within line.

Examples: a, span, img, strong

Slide 10 of 30

Introduction to Networking Basics

Networking connects computers to share data and resources.

  • Fundamental to the internet and web browsing
  • Web uses protocols to transfer and display content
Slide 11 of 30

What are Protocols?

Protocols are rules that define how data is transmitted over a network.

  • Common protocol: HTTP (Hypertext Transfer Protocol)
  • Secure variant: HTTPS (HTTP Secure)
Slide 12 of 30

HTTP - Hypertext Transfer Protocol

Protocol for communication between web browsers and servers.

  • Works over TCP/IP for web page delivery
  • Stateless protocol: no data retention between requests
Slide 13 of 30

HTTPS - HTTP Secure

Secure communication over the internet using SSL/TLS encryption.

  • Protects data integrity and privacy
  • Uses certificates to validate identity
  • Important for safe online transactions
Slide 14 of 30

How HTTPS Works

  • Browser connects to server and requests certificate
  • Server sends SSL certificate to browser
  • Browser verifies certificate authenticity
  • Encrypted session established for data transfer
Slide 15 of 30

What is a Browser?

A software application to access and display web content.

  • Examples: Chrome, Firefox, Edge, Safari
  • Interprets HTML, CSS, and JavaScript
Slide 16 of 30

How Browsers Work

  • User types URL or clicks link
  • Browser sends HTTP/HTTPS request to web server
  • Server responds with HTML data
  • Browser renders the page content
Slide 17 of 30

URLs and Domains

URL: Web address that locates a resource

Domain: Name representing an IP address

Example: https://www.example.com/index.html

Further HTML Elements

  • div - section container
  • span - inline container
  • form, input - user input elements
  • table - tabular data
Slide 22 of 30

Useful HTML Attributes

  • alt - alternative text for images
  • title - tooltip text
  • target - link target (_blank, _self)

Slide 23 of 30

Using Comments Effectively

Helpful for collaborators and future maintenance.

Never leave sensitive information in comments!

Slide 24 of 30

Common HTML Mistakes

  • Missing closing tags
  • Improper nesting of tags
  • Incorrect attribute values
  • Using deprecated tags
Slide 25 of 30

Web Security Basics

Browsers warn about insecure websites (HTTP without HTTPS).

Always use HTTPS for sensitive data exchange.

Slide 26 of 30

How to Check if a Website is Secure

  • Look for a padlock icon in address bar
  • URL starts with https://
  • Validate certificate details if needed
Slide 27 of 30

Tips for Writing Clean HTML

  • Indent your code properly
  • Use lowercase for tags and attributes
  • Add meaningful comments
  • Validate HTML with online tools
